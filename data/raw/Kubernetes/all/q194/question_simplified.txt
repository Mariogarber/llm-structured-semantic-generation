Modify the two pods in the Kubernetes manifest to avoid conflicts with the replicaset and ensure successful creation of 3 replicas without any non-homogenous Pods running mongoDB.

```
apiVersion: v1
kind: Pod
metadata:
  name: pod1
  labels:
    tier: backend
spec:
  containers:
  - name: hello1
    image: gcr.io/google-samples/hello-app:2.0
---
apiVersion: v1
kind: Pod
metadata:
  name: pod2
  labels:
    tier: backend
spec:
  containers:
  - name: hello2
    image: gcr.io/google-samples/hello-app:1.0
---
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: backend
  labels:
    app: database
    tier: backend
spec:
  replicas: 3
  selector:
    matchLabels:
      tier: backend
  template:
    metadata:
      labels:
        tier: backend
    spec:
      containers:
      - name: db-container
        image: mongo:latest
```

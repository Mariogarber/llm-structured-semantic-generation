修改 Kubernetes 清单中的两个 Pod，以避免与 ReplicaSet 冲突，并确保成功创建 3 个副本，而不运行任何非同质的运行 mongoDB 的 Pod。

```
apiVersion: v1
kind: Pod
metadata:
  name: pod1
  labels:
    tier: backend
spec:
  containers:
  - name: hello1
    image: gcr.io/google-samples/hello-app:2.0
---
apiVersion: v1
kind: Pod
metadata:
  name: pod2
  labels:
    tier: backend
spec:
  containers:
  - name: hello2
    image: gcr.io/google-samples/hello-app:1.0
---
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: backend
  labels:
    app: database
    tier: backend
spec:
  replicas: 3
  selector:
    matchLabels:
      tier: backend
  template:
    metadata:
      labels:
        tier: backend
    spec:
      containers:
      - name: db-container
        image: mongo:latest
```
